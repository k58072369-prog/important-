تعليمات إلزامية قبل التنفيذ

هذا المشروع عبارة عن نظام إدارة لمكتب تحفيظ القرآن الكريم يعمل كتطبيق Desktop App.

المشروع موجود بالفعل ويعمل حالياً.

المطلوب هو:

- تطوير النظام.
- تحسين النظام.
- إضافة المميزات الجديدة.
- إصلاح الأخطاء.
- رفع الأداء والاستقرار.

فقط.

ويُمنع منعاً باتاً:

- حذف أي قسم موجود.
- حذف أي صفحة موجودة.
- حذف أي وظيفة موجودة.
- حذف أي بيانات محفوظة.
- إعادة بناء المشروع من الصفر.
- استبدال الأقسام الحالية.

ويجب الحفاظ على جميع البيانات الحالية كما هي.

---

أولاً: إنشاء نظام كود تلقائي فريد لكل طالب

يجب إنشاء نظام أكواد احترافي للطلاب.

عند إضافة أي طالب جديد:

يقوم النظام تلقائياً بإنشاء كود فريد وغير مكرر.

أمثلة:

STD-00001

STD-00002

STD-00003

أو أي صيغة احترافية مشابهة.

شروط الكود:

- لا يتكرر نهائياً.
- مرتبط بالطالب مدى الحياة.
- لا يتغير عند تعديل البيانات.
- يظهر داخل ملف الطالب.
- يظهر داخل التقارير.
- يظهر داخل ملفات PDF.
- يظهر داخل ملفات HTML.
- يظهر داخل تقارير المسابقات.
- يظهر داخل نتائج البحث.
- يمكن البحث بواسطة الكود.

---

ثانياً: تطوير نظام التقارير الشهرية للطلاب بالكامل

يجب إنشاء نظام تقارير شهرية احترافي متكامل.

لكل طالب تقرير مستقل شهري.

أمثلة:

- تقرير يناير 2026.
- تقرير فبراير 2026.
- تقرير مارس 2026.

ويتم إنشاؤه تلقائياً اعتماداً على البيانات الفعلية داخل النظام.

التقرير النصي التفصيلي

إنشاء تقرير نصي تلقائي اعتماداً على البيانات الفعلية فقط.

بدون أي ذكاء اصطناعي.

مثال:

حقق الطالب خلال هذا الشهر مستوى جيداً من الالتزام بالحضور، حيث بلغت نسبة حضوره 92%، كما أحرز تقدماً ملحوظاً في الحفظ مقارنة بالشهر السابق، وارتفع متوسط تقييمه بمقدار 15%، مع استمرار جيد في المراجعة الدورية.

ويتم إنشاء النص بواسطة قواعد برمجية ومنطقية فقط.

خامساً: إنشاء ملف PDF واحد شامل لجميع تقارير الطلاب

المطلوب إنشاء ملف PDF واحد فقط يحتوي على:

جميع تقارير الطلاب للشهر المحدد.

مثال:

تقرير شهر مايو 2026.

يتم إنشاء ملف واحد:

Students_Report_May_2026.pdf

ويحتوي على:

- صفحة لكل طالب.
- كود الطالب.
- بياناته.
- حضوره.
- غيابه.
- حفظه.
- مراجعته.
- تقييماته.
- التقرير النصي الخاص به.

ويتم ترتيب الصفحات تلقائياً.

---

مواصفات ملف HTML

يجب أن:

- يعمل بدون إنترنت.
- يحتوي CSS داخلي.
- يحتوي JavaScript داخلي عند الحاجة.
- لا يعتمد على CDN.
- لا يعتمد على مواقع خارجية.
- لا يعتمد على مكتبات إنترنت.

ويكون قابلاً للحفظ على الجهاز.

وقابلاً للفتح من أي جهاز.

وقابلاً للطباعة مباشرة.
