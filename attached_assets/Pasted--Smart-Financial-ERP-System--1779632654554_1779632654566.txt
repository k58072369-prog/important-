💰 برومبت نظام مالي عملاق ذكي متكامل (Smart Financial ERP System)

المطلوب تصميم وتطوير نظام مالي احترافي عملاق داخل نظام إدارة شامل (طلاب – معلمين – حلقات – حصص – متابعة – استمارات)، بحيث يكون النظام المالي ذكي، مترابط، وقابل للتوسع، ويعتمد على قاعدة بيانات قوية محلية تتحمل بيانات ضخمة جدًا بدون بطء أو فقدان بيانات.

━━━━━━━━━━━━━━

🧠 الفكرة الأساسية للنظام

تحويل النظام المالي إلى نظام ERP ذكي متكامل يقوم على:
• تسجيل كل العمليات المالية تلقائيًا
• ربط كل حركة مالية بباقي النظام (طلاب / معلمين / حلقات / حصص)
• إنشاء استمارات مالية ديناميكية (Forms)
• تحليل الأرباح والخسائر بشكل لحظي
• تقليل الإدخال اليدوي لأقصى حد

━━━━━━━━━━━━━━

💾 قاعدة البيانات (Core Engine)

• قاعدة بيانات محلية قوية (SQLite / PostgreSQL محلي)
• دعم بيانات ضخمة جدًا بدون بطء
• علاقات قوية بين كل الجداول
• حفظ تلقائي لكل العمليات (إضافة / تعديل / حذف)
• دعم النسخ الاحتياطي والاسترجاع
• منع التكرار والتضارب في البيانات
• تصميم قابل للتوسع المستقبلي (Scalable Architecture)

━━━━━━━━━━━━━━

📋 نظام الاستمارات المالية (Financial Forms System)

نظام مرن لإنشاء استمارات مخصصة للمصروفات والورادات:

━━━━━━━━━━━━━━

💸 1) استمارة المصروفات (Expense Form Builder)

📌 يمكن إنشاء استمارة مصروف مخصصة تحتوي على:

• اسم المصروف
• التصنيف (إيجار / أدوات / صيانة / رواتب / أخرى)
• المبلغ
• التاريخ
• طريقة الدفع
• مرتبط بـ (معلم / حلقة / عام / إداري)
• ملاحظات

📌 مميزات:
• إمكانية إضافة أو تعديل أي استمارة
• حفظ تلقائي داخل قاعدة البيانات
• إمكانية إنشاء قوالب مصروفات جاهزة
• تتبع تاريخ كل تعديل على الفاتورة

━━━━━━━━━━━━━━

📥 2) استمارة الورادات (Income Form)

• اشتراكات الطلاب
• تبرعات
• دخل إضافي

📌 عند إنشاء فاتورة شهرية:
• يتم توليد استمارة تلقائية لكل الطلاب
• عرض حالة الدفع لكل طالب (مدفوع / غير مدفوع / جزئي)
• إمكانية تسجيل الدفع مباشرة وربطه بالطالب

━━━━━━━━━━━━━━

🎁 3) استمارات مخصصة (Custom Forms)

• إمكانية إنشاء أي استمارة مالية جديدة
• تحديد الحقول يدويًا
• ربطها بأي جزء من النظام
• حفظها كنموذج قابل لإعادة الاستخدام

━━━━━━━━━━━━━━

📊 المعاملات المالية (Transactions System)

كل العمليات يتم تحويلها تلقائيًا إلى معاملات:

• مصروفات
• ورادات
• تحويلات داخلية

📌 كل معاملة تحتوي على:
• النوع
• القيمة
• التاريخ
• التصنيف
• الرابط (طالب / معلم / حلقة / عام)
• سجل التعديل الكامل

━━━━━━━━━━━━━━

👨‍🎓 الورادات الذكية (Smart Billing System)

• إنشاء فواتير شهرية تلقائيًا لكل الطلاب
• عرض جميع الطلاب في نظام دفع موحد
• حالات الدفع:

- مدفوع
- غير مدفوع
- جزئي

📌 عند الدفع:
• تسجيل مباشر في الورادات
• تحديث حالة الطالب فورًا
• ربط العملية بملفه المالي

━━━━━━━━━━━━━━

👨‍🏫 الرواتب الذكية للمعلمين (Smart Payroll Engine)

النظام يحسب الرواتب تلقائيًا بدون تدخل يدوي:

📌 طرق الحساب:
• عدد الحصص
• نسبة من الورادات
• مبلغ ثابت
• نظام مختلط

📌 النظام يقوم بـ:
• حساب الراتب تلقائيًا
• تسجيله كمصروف “رواتب”
• ربطه بالحلقات والحصص تلقائيًا
• تحديثه حسب النشاط والأداء

━━━━━━━━━━━━━━

📊 قسم الأرباح والتحليلات (AI Financial Dashboard)

نظام تحليلي ذكي بالكامل:

📈 مؤشرات:
• إجمالي الورادات
• إجمالي المصروفات
• صافي الربح الفعلي

📉 تحليلات متقدمة:
• أرباح شهرية
• أداء كل معلم ماليًا
• تكلفة الطالب
• أكثر الحلقات ربحًا
• نسبة التحصيل
• تحليل الاتجاهات المالية

━━━━━━━━━━━━━━

🔗 الربط الذكي مع النظام بالكامل

• الطالب → اشتراكات + مدفوعات
• المعلم → رواتب + أداء
• الحلقة → دخل + تكلفة تشغيل
• الحصة → إنتاجية وتأثير مباشر

📌 أمثلة:
• دفع طالب → يتحول تلقائيًا إلى وارد مالي
• زيادة حصص معلم → زيادة راتبه تلقائيًا
• أي مصروف → يؤثر مباشرة على الأرباح
• كل عملية مرتبطة بسجلها الأساسي

━━━━━━━━━━━━━━

⚙️ خصائص النظام الذكي العملاق

• نظام تلقائي بالكامل بدون إدخال متكرر
• سرعة عالية جدًا حتى مع بيانات ضخمة
• حفظ فوري بدون فقدان بيانات
• قابل للتوسع (ERP كامل مستقبلًا)
• تقارير لحظية دقيقة
• سجل تاريخي لكل العمليات
• واجهة مرنة وقابلة للتطوير

━━━━━━━━━━━━━━

🎯 الهدف النهائي

بناء نظام مالي عملاق ذكي (Enterprise Level System):
• ERP متكامل
• نظام استمارات مرن
• محاسبة ذكية تلقائية
• تحليلات مالية لحظية
• ربط كامل بكل أجزاء النظام
• أداء قوي ومستقر وقابل للتوسع
