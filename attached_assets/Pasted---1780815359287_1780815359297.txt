إصلاح عاجل لمشكلة الشاشة البيضاء بعد التعديلات

المشكلة الحالية

تم تنفيذ التعديلات بنجاح داخل المشروع، ولا توجد أخطاء ظاهرة أثناء عملية التطوير، ولكن عند الضغط على:

- Run
- Preview
- View App

لا يتم عرض النظام.

وتظهر شاشة بيضاء فارغة بالكامل.

المطلوب هو تنفيذ فحص تقني شامل وعميق للمشروع بالكامل لتحديد السبب الحقيقي وإصلاحه بشكل نهائي.

---

أولوية قصوى

لا تقم بإضافة أي ميزة جديدة حالياً.

لا تقم بتعديل تصميم النظام.

لا تقم بحذف أي قسم.

لا تقم بإعادة بناء المشروع من الصفر.

المطلوب فقط:

- اكتشاف سبب الشاشة البيضاء.
- إصلاح السبب.
- إعادة تشغيل المشروع بشكل طبيعي.

---

المطلوب تنفيذه بالتفصيل

المرحلة الأولى: فحص Console بالكامل

افحص جميع الأخطاء داخل:

- Browser Console
- Developer Tools
- Network Tab

وابحث عن:

- JavaScript Errors
- TypeScript Errors
- React Errors
- Runtime Errors
- Import Errors
- Build Errors
- Hydration Errors
- Module Errors

وأظهر جميع الأخطاء المكتشفة.

---

المرحلة الثانية: فحص نقطة البداية

تحقق من:

- main.tsx
- main.jsx
- index.tsx
- index.jsx
- App.tsx
- App.jsx

وتأكد من:

- أن App يتم تحميله فعلياً.
- أن React يتم عمل Render له.
- أن Root Element موجود.
- أن التطبيق لا يتوقف أثناء التحميل.

---

المرحلة الثالثة: فحص الاستيرادات Imports

افحص جميع:

import

داخل المشروع.

وتأكد من عدم وجود:

- ملف محذوف.
- مسار خاطئ.
- Component غير موجود.
- Hook غير موجود.
- Context غير موجود.
- Store غير موجود.

أي Import مكسور قد يؤدي إلى شاشة بيضاء.

---

المرحلة الرابعة: فحص Routes

إذا كان المشروع يستخدم:

- React Router
- TanStack Router
- Next Router

فيجب التحقق من:

- جميع المسارات.
- الصفحات.
- Layouts.
- Guards.
- Providers.

والتأكد من أن الصفحة الرئيسية يتم تحميلها فعلياً.

---

المرحلة الخامسة: فحص المكونات الجديدة

بما أن المشكلة ظهرت بعد التعديلات الأخيرة.

قم بفحص جميع المكونات التي تمت إضافتها أو تعديلها مؤخراً وخاصة:

- شاشة الفيديو الافتتاحية.
- نظام الصوت.
- نظام الأذكار.
- نظام المسابقات.
- نظام الدورات.
- التخزين المحلي.
- لوحة الصدارة.
- الإدارة المالية.

وابحث عن أي:

- Infinite Render Loop.
- State Loop.
- Hook Error.
- Undefined Variable.
- Null Reference.
- Async Error.

---

المرحلة السادسة: فحص IndexedDB و Local Storage

قد يكون المشروع يتوقف أثناء تحميل البيانات.

لذلك:

تحقق من:

- IndexedDB.
- LocalStorage.
- Cache Storage.

وتأكد من:

- عدم وجود بيانات تالفة.
- عدم وجود Migration Error.
- عدم وجود Parsing Error.
- عدم وجود JSON Error.

وفي حال اكتشاف بيانات تالفة:

قم بإصلاحها أو تجاهلها دون كسر النظام.

---

المرحلة السابعة: فحص Providers

افحص:

- Context Providers
- Theme Providers
- Auth Providers
- Store Providers

وتأكد أن جميعها تعمل بشكل صحيح.

---

المرحلة الثامنة: فحص Network Requests

إذا كان يوجد أي طلب:

- API
- Fetch
- Axios

فتأكد من أن فشل الطلب لا يؤدي إلى انهيار التطبيق.

يجب أن يعمل النظام حتى عند فشل الطلبات.

---

المرحلة التاسعة: تفعيل Error Boundary

قم بإضافة Error Boundary عالمي.

بحيث:

إذا تعطل أي Component:

- لا تظهر شاشة بيضاء.
- تظهر رسالة خطأ واضحة.
- يتم تسجيل الخطأ.

---

المرحلة العاشرة: تحديد المكون المسبب للمشكلة

قم بتعطيل تحميل المكونات تدريجياً.

مثال:

1. تشغيل App فقط.
2. تشغيل Layout فقط.
3. تشغيل Dashboard فقط.
4. تشغيل الأقسام واحداً واحداً.

حتى يتم اكتشاف المكون المسؤول عن الشاشة البيضاء.

---

فحص البناء Build

نفذ:

- npm run build
- npm run dev

وأصلح جميع الأخطاء والتحذيرات الحرجة.

يجب أن تكون نتيجة البناء ناجحة 100%.

---

فحص الأداء والذاكرة

تحقق من عدم وجود:

- Memory Leaks
- Infinite Loops
- Excessive Re-renders
- Blocking Operations

التي قد تمنع تحميل الصفحة.

---

المطلوب بعد انتهاء الفحص

أريد تقريراً نهائياً يحتوي على:

1. السبب الحقيقي للشاشة البيضاء.
2. الملف المسؤول عن المشكلة.
3. السطر أو الجزء المسبب للمشكلة.
4. ما الذي تم إصلاحه.
5. صورة أو لقطة للشاشة الأولى التي تظهر بعد نجاح التشغيل.
6. تأكيد أن التطبيق يعمل بشكل طبيعي.
7. تأكيد أن جميع الأقسام ما زالت موجودة.
8. تأكيد أن أي بيانات لم تُحذف.

---

شرط نجاح المهمة

لا تعتبر المهمة منتهية إلا إذا:

- اختفت الشاشة البيضاء بالكامل.
- تم فتح المشروع بنجاح.
- ظهرت الصفحة الرئيسية.
- تم اختبار جميع الأقسام.
- تم التأكد من عدم وجود أخطاء حرجة.
- تم التأكد من أن المشروع يعمل بشكل كامل دون انهيار.

الهدف النهائي: إعادة المشروع للعمل بشكل طبيعي، اكتشاف السبب الحقيقي للشاشة البيضاء، إصلاحه نهائياً، والتأكد من أن جميع التعديلات السابقة ما زالت تعمل دون فقدان أي بيانات أو أقسام.
