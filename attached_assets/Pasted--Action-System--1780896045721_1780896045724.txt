تمام، دي نسخة برومت قوية ومتكاملة تشمل كل الاقتراحات ما عدا نظام الـ Action System الموحد زي ما طلبت:


---

🔥 برومت تطوير نظام احترافي Offline كامل (بدون إنترنت + حماية بيانات قصوى + استقرار عالي)

تعليمات إلزامية ونهائية

هذا المشروع عبارة عن Desktop / Web Application يعمل بالكامل بدون إنترنت (Offline 100%) ويجب أن يكون نظاماً احترافياً شديد الاستقرار وقابل للتوسع لسنوات طويلة.

⚠️ مهم جداً:

ممنوع الاعتماد على أي API خارجي.

ممنوع أي خدمة سحابية.

ممنوع أي ذكاء اصطناعي أو AI Modules.

ممنوع إرسال أو استقبال أي بيانات خارج الجهاز.

النظام يجب أن يعمل بالكامل محلياً فقط.



---

أولاً: نظام الإقلاع والتشغيل (Boot System)

عند فتح المشروع يجب تنفيذ تسلسل إجباري:

1. شاشة تحميل احترافية (Loading Screen).


2. فحص صحة النظام (System Health Check).


3. فحص سلامة البيانات (Data Integrity Check).


4. التحقق من المسارات والصفحات (Route Validation).


5. في حال وجود خطأ:

يتم تشغيل وضع آمن (Safe Mode)

عرض سبب المشكلة

إتاحة إعادة التشغيل دون فقدان البيانات




⚠️ ممنوع ظهور شاشة بيضاء نهائياً.


---

ثانياً: منع الشاشة البيضاء نهائياً

يجب معالجة أي حالة خطأ داخل النظام بحيث:

لا يتم كسر الواجهة.

لا يتم توقف التطبيق.

لا يتم عرض صفحة فارغة.


بدلاً من ذلك:

عرض واجهة بديلة (Fallback UI)

تسجيل الخطأ في سجل النظام

محاولة إعادة التحميل تلقائياً



---

ثالثاً: نظام التخزين المحلي القوي جداً (أولوية قصوى)

يجب الاعتماد الكامل على:

IndexedDB (قاعدة البيانات الأساسية)

LocalStorage (الإعدادات السريعة)

Cache Storage (للملفات)


آلية التخزين الإجباري:

أي عملية (إضافة / تعديل / حذف / أرشفة) يتم حفظها فوراً.

بدون زر حفظ يدوي.

بدون تأخير.


نظام الحماية من فقدان البيانات:

يجب ضمان عدم فقدان أي بيانات عند:

إغلاق البرنامج فجأة

إعادة تشغيل الجهاز

انقطاع الكهرباء

كراش التطبيق

تحديث الصفحة



---

رابعاً: نظام النسخ الاحتياطي والاستعادة

يجب إنشاء:

Auto Backup System

نسخ احتياطي تلقائي دوري.

حفظ أكثر من نسخة.

إدارة نسخ احتياطية متعددة.


Auto Recovery System

استعادة تلقائية عند فتح النظام.

الرجوع لآخر حالة مستقرة.

إصلاح الأخطاء البسيطة تلقائياً.
